Detailed Description

Utility class aimed to correctly initialize Logging into log4net.

It is important to use this class in .NET Core projects. .NET Core does not have support of System.Configuration, but log4net tries to access System.Configuration API, which is resulted in bunch of exceptions, logged into Game Carrier log file. In order to avoid this, perform following steps:

  • create internal class EntryPoint, derived from GcEntryPointBase and put it into Service Application project
  • put EntryPoint class into empty namespace

  • within .ctor of EntryPoint make a call to GameCarrier.Common.Log4net.Log4netHelper.Configure();

    internal class EntryPoint : GcEntryPointBase { public EntryPoint() { GameCarrier.Common.Log4net.Log4netHelper.Configure(); } }

static void GameCarrier.Common.Log4net.Log4netHelper.Configure ( string  configFileName = "log4net.config")
static

Configures log4net from configuration file (default is log4net.config), located in directory of calling assembly (directory of Service Application). Following log4net Properties configured:

  • ApplicationDirectory - directory of calling assembly
  • ApplicationLogPath - directory of calling assembly
  • ApplicationPath - path to Service Application assembly (without extension)

log4net Properties can be used in config file in following way: 

<file type="log4net.Util.PatternString" value="%property{ApplicationPath}.log" />

<file type="log4net.Util.PatternString" value="%property{ApplicationLogPath}\\PhotonAdapterTestApp.log" />
