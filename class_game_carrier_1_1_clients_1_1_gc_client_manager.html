Detailed Description

Client Manager class that operate with client-system

  • Init(GcClientMode) - configure client-system mode
  • Service() - force client-system message loop (send/receive messages)
  • Cleanup() - cleanup client-system

◆ Init()

void GameCarrier.Clients.GcClientManager.Init ( GcClientMode  mode)

Configure client-system mode. Client-system mode configured globally for client application, meaning all clients of application will work under same mode. Client mode assumes concrete thread model + code usage approach. In order to re-initialize client-system - make a Cleanup() call befone call to Init() with new mode.

  • Passive mode - Sends & receives performed under call to GcClientManager.Instance.Service(). Calls to GcClientManager.Instance.Service() are not thread-safe.
  • Active mode - Sends & receives performed immediatelly from IO thread. No need to call GcClientManager.Instance.Service().
  • Hybrid mode - Sends performed immediatelly from IO thread. Receives performed under call to GcClientManager.Instance.Service(). Calls to GcClientManager.Instance.Service() are thread-safe.
