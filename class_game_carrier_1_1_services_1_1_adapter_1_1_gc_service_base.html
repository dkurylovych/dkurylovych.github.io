Detailed Description

Base class of Service When develop new service - inherit your service class from GcServiceBase, and:

  • provide ServiceName in constructor
  • create class for service connection
  • override OnConnect method and return new instance of connection
  • override ConfigureContainer method to initialize service interface client

◆ OnStart()

override void GameCarrier.Services.Adapter.GcServiceBase.OnStart ( )

Callback method executed on Service start. During Start phase following steps performed:

  • ConfigureContrainer()
  • ConfigureService()
  • Connect to Discovery Service
  • Promote Public Endpoint of service to Discovery Service
  • Service becomes available for Clients over Public Endpoint

Developer should configure public_endpoint in GameCarrier's config.json file (for application): 

"name": "Persistence",
"type_name": "GameCarrier.Persistence.Service.PersistenceService",
"file_name": "%SOURCE_DN%/managed/NetServices/GC.Services.TestApp/bin/%BUILD_TYPE%/%PLATFORM%/GC.Persistence.Service.dll",
"public_endpoint": "wss://localhost:7681/Persistence",
"discovery_endpoint": "wss://localhost:7700/Discovery"

◆ OnStop()

override void GameCarrier.Services.Adapter.GcServiceBase.OnStop ( )

Callback method executed on Service stop.

  • ServiceFactory.Dispose() is executed here, causing disconnect from Discovery Service.
  • Disconnect from discovery Service means that Clients will not be able to discover/connect to Service
